## 五、会话模型的实际应用

### 当前时刻判断（如何用会话状态修正µ）

1. 查看当天已过时段内每个会话是否发生
2. 缺席会话 → 从µ中下移对应条数（见下表）
3. 评估剩余时段的期望推文数
4. 最终µ = 当前推文数 + 剩余期望

| 缺席会话 | 北京时间 | µ下修 |
|---------|---------|------|
| 深夜会话缺席 | 11点–17点 | **-14条** |
| 下午会话缺席 | 0点–6点   | **-14条** |
| 上午会话缺席 | 20点–24点 | **-11条** |
| 傍晚会话缺席 | 6点–11点  | **-11条** |

### 连续降温信号

若连续2天发推量低于历史均值30%+：
- 视为µ下修信号
- 必须在当天死区重算µ并评估调仓
- **不能等"再看一天"**

## 六、推文内容风格（@rich_adul 内容运营参考）

> 本节是博主推文风格指南，用于日常内容创作，不是交易规则。

### 高互动推文规律

**结论：故事 > 信息。数字开头 > 文字开头。他人故事 > 自我推销。**

| 推文开头类型 | 互动效果 |
|-----------|---------|
| 数字+意外结果（"500万美金，一夜归零"） | 🔥🔥 最高 |
| 情报型独家解读 | 🔥 高 |
| 故事型反转 | 🔥 高 |
| 链接型无叙述 | 🔵 低 |

### 最有效的开头公式

**10字以内，制造强烈好奇心**：

```
✅ 500万美金，一夜归零。
✅ 18美金变3800美金。
✅ 1.5%的概率翻盘！
✅ 马斯克今天突然安静了。

❌ 马斯克推文预测市场更新
❌ 本周预测市场数据汇总
```

### 不同场景的推文开头

| 场景 | 推荐开头 |
|------|---------|
| 马斯克今天发推异常少 | 「马斯克今天突然安静了。」 |
| 马斯克最后两天爆发 | 「他开始补了。」 |
| 某区间赔率明显低估 | 「有个区间，市场给的概率只有 X%，但我的模型说是 Y%。」 |
| 预测落点接近边界 | 「还差 X 条，就跨区间了。」 |
| 结算当天 | 「今晚 24:00，答案揭晓。」 |
| 押中了 | 「押中了。」（就这四个字） |
| 亏了 | 「翼仓没撑住。」 |

### 语言风格

**用：** 「但是——」「然而」「有趣的是」「我不这么看」「我买了一点」「明天看看」「有人」

**不用：** 「数据更新」「汇总」「播报」「建议购买」多余hashtag、Emoji滥用
