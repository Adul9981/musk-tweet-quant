# 商业化路线图

> 记录产品从工具到商业产品的演化方向和核心想法。

---

## 一、商业模式

### 路线 A：订阅制
- 用户按月/季/年付费，解锁完整功能
- 适合：有稳定交易习惯的用户，愿意为信息优势付费
- 定价方向待定（参考同类预测市场工具：$10–$50/月）
- 免费层保留基础赔率查看，付费层解锁：µ预测、入场建议、Telegram 预警

### 路线 B：金库制（自动化交易）
- 用户存入资金，系统根据知识库规则自动判断入场、下注、出场
- 本质是"策略即产品"，用户只需充值和看结果
- 技术门槛高：需要对接 Polymarket 链上合约，自动执行买卖
- 风险合规问题需要提前研究（自动化投注平台的法律边界）
- 商业模式：按盈利分成（如净利润的 10–20%）

## 二、产品简化原则

> 核心判断：现有工具功能太多，用户实际用不到。商业化版本要砍掉噪音，只留关键路径。

### 用户真正需要的（保留）
1. **当前期推文进度** — 已发多少条、今日新增、距结束还有多久
2. **各区间赔率** — 当前赔率一眼看清，配合 µ预测落点
3. **入场建议** — 明确告诉用户"现在该不该进、进哪个区间"
4. **预警通知** — 关键节点主动推送，不需要用户盯着屏幕

### 可以隐藏或删减的（简化）
- 发推热力图（高级用户才用，移到「高级」标签或删除）
- 推文生成（和核心交易决策无关）
- 策略指南（内容折叠，默认收起）
- 持仓管理（可以简化为"我的仓位"一行）

### 简化后的导航结构（初稿）
```
首页（市场概览 + 入场建议）
├── 当前期状态（推文进度条 + 赔率）
├── 操作建议（买哪个区间 / 等待 / 不操作）
└── 我的仓位（简单记录）

通知设置
└── Telegram / 站内通知配置
```

## 三、通知与预警功能改进方向

> 目标：消息要让用户一眼看懂，不需要理解背景知识就能行动。

### 现状问题
- Telegram 预警内容信息密度过高，新用户看不懂
- 需要手动配置 Bot Token 和 Chat ID，门槛高
- 没有分级（普通提醒 vs 紧急信号）

### 改进方向
1. **消息模板简化** — 每条预警只说三件事：
   - 发生了什么（"马斯克今日已发 X 条"）
   - 对当前市场意味着什么（"µ落点偏向 140–159 区间"）
   - 建议动作（"可考虑入场 / 继续观察 / 注意止损"）

2. **降低配置门槛** — 提供一键绑定 Telegram 的引导流程（扫码/点链接），不让用户手动填 Token

3. **分级通知**
   - 🟢 常规播报（每日定时推送发推进度）
   - 🟡 入场信号（满足入场条件时触发）
   - 🔴 紧急预警（死区临近 / 异常波动）

4. **多渠道支持**（长期）
   - 当前：Telegram
   - 后续考虑：微信公众号推送、邮件、站内消息

## 四、阶段规划（初稿）

| 阶段 | 目标 | 关键动作 |
|------|------|---------|
| 0（当前） | 工具可用，数据稳定 | 修复数据源、稳定运行 |
| 1（产品化） | 简化界面，新用户能上手 | 精简导航、优化通知模板 |
| 2（订阅制） | 跑通付费流程 | 接入支付（Stripe/加密支付）、设计免费/付费分层 |
| 3（金库制） | 自动化交易 | 对接 Polymarket API、设计策略执行引擎 |
