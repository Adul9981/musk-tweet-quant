# 商业化规划

> 产品方向：从个人工具 → 简单易用的付费预测辅助平台

---

## 一、商业化路线

### 路线1：订阅制
- 用户按月/年付费，解锁高级功能
- 适合：有稳定使用需求的预测市场用户
- 定价参考：$9–$29/月（基础/专业档）

**免费用户可用：**
- 市场概览、赔率展示
- 基本时间轴

**付费用户解锁：**
- 实时推文计数（twitterapi.io 精准源）
- Telegram 预警推送
- 入场建议（价值比 + µ预测）
- 历史回测数据查看

---

### 路线2：金库制（自动化交易）
- 用户存入资金，系统按策略自动在 Polymarket 下注
- 盈利后收取分成（如 20% 业绩提成）
- 风险高，需要：合规审查 + 链上钱包集成 + 风控模块

**关键依赖：**
- Polymarket CLOB API（程序化下单）
- 链上签名（Polygon 钱包）
- 策略引擎（基于知识库 05_入场与仓位框架）

**优先级：** 订阅制先行，金库制作为 v2 目标

---

## 二、产品简化原则

> 现状问题：页面 Tab 太多（市场概览/概率分析/持仓管理/发推热力图/推文生成/策略指南/Telegram预警），用户不知道该用哪个。

### 简化方向

**保留（核心功能）：**
- 📊 当前市场：推文数量进度、剩余时间、赔率
- 🔔 预警：入场时机提醒（Telegram + 网页通知）
- 📈 入场建议：价值比 + 推荐区间（一键看结论）

**收起/精简：**
- 发推热力图 → 折叠进"分析"模块
- 推文生成 → 移入"高级工具"
- 持仓管理 → 订阅用户专属

**目标 UI 结构（三屏设计）：**
```
屏幕1（首页）：当前期进度 + 推文计数 + 赔率
屏幕2（入场）：建议区间 + 价值比 + 入场/出场按钮
屏幕3（预警）：Telegram 绑定 + 预警规则设置
```

---

## 三、预警与通知优化

> 现状问题：Telegram 预警配置复杂，普通用户难以独立完成。

### 优化方向
- **网页内一键绑定 Telegram**：输入 Chat ID 即完成，不需要手动改配置
- **预警级别简化为三档：**
  - 🟢 低优先级：每日早报（8:00 BJ 推送前日总结）
  - 🟡 中优先级：入场窗口提醒（到期前 2–2.5 天触发）
  - 🔴 高优先级：极端事件（马斯克沉默超 8h / 推文速率异常）
- **微信/邮件通知**作为备选渠道（订阅制功能）

| 阶段 | 目标 | 关键动作 |
