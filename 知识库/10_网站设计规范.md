# 网站设计规范

> 记录 2026-06-02 设计讨论结果，执行前必须对照此文档。

---

## 整体风格

- **风格**：现代 App，不是炫技，不是信息堆砌
- **原则**：不同颜色 = 不同信号，颜色有含义，不是装饰
- **文字**：能用数字+标签表达的，不写长句

---

## 颜色体系

| 颜色 | Tailwind | 含义 | 使用场景 |
|------|----------|------|---------|
| 🟢 亮绿 | emerald-400/500 | 买入 · 盈利 · 机会 | 推荐买入、正收益、活跃窗口、高VR区间 |
| 🔴 红 | rose-400/500 | 卖出 · 亏损 · 警告 | 止损、死区提示、VR<1 |
| 🟡 黄 | amber-400/500 | 注意 · 边界风险 | 区间边界信号、出本留利提醒 |
| ⚪ 白/浅灰 | slate-100/200 | 核心数字 | 推文数、概率、价格 |
| 🔵 青绿 | teal-400 | 辅助信息 | 时间、次要数据、标签 |
| 暗绿边框 | emerald-900/800 | 卡片分层 | 区分卡片层次，不用颜色堆砌 |
- **背景**：#0d0d0d（近黑）
- **卡片**：#141414（深灰）
- **卡片边框**：emerald-900/30 或 slate-700/40

## 导航标签

- 前3个（核心）：市场概览 · 概率分析 · 发推热力图 → 高亮显示
- 后4个（辅助）：持仓管理 · 推文生成 · 策略指南 · 预警 → 低调灰色
- 分隔线区分两组

## 市场概览页面布局

### 顶部：市场切换标签（小，放右上角）
```
[ 5月26日–6月2日 ]  [ 5月29日–6月5日 ]  [ 6月2日–6月9日 ]
```
- 标签小而紧凑，当前选中的高亮绿色
- 日期格式必须完整：**X月X日–X月X日**（补"日"字）

### 第一行：4个核心数字
```
推文数 142   时速 21/天   剩余 16h   🔥 深夜爆发期
```
- 数字大，标签小
- 窗口期用 emoji + 文字 + 对应颜色

### 第二行：预测信息
```
预测落点 ~155条   精准度 82%（距到期16h）
```
- 精准度随剩余时间动态变化（来自µ不确定性表）

### 第三行：左右分区
**左：本期每日发推**
- 热力色块行（7个格子，颜色深浅=数量多少）
- 格子下方显示具体数字
- 悬停显示具体日期

**右：核心区间定价**
- 显示3-5个核心区间
- 每行：区间名 · 价格 · VR · 盈亏比
- VR≥1.5 的区间标 ⭐，VR≥2.0 标 ⭐⭐

### 底部：入场信号条
```
💡 120–139 低价机会 · VR 1.8 · 盈亏比 3.6x · 建议仓位 ≤20%
```
- 只有真正有信号时才显示（VR≥1.5 且价格≤20¢）
- 无信号时不显示，不占空间

| 距到期 | µ误差 | 区间精准度 |
|--------|-------|----------|
| 3天 | ±50条 | ~45% |
| 2天 | ±35条 | ~58% |
| 1天 | ±20条 | ~72% |
| 12h | ±12条 | ~82% |
| 6h | ±8条 | ~88% |

## 入场信号重定义

> 旧规则「到期前2-2.5天入场」存在问题：此时中心区间可能已达40¢+，VR<1。
> 新规则：时间 × 价格 双重条件。

**触发条件（同时满足）：**
1. VR ≥ 1.2（价格合理）
2. 目标区间价格 ≤ 30¢（中心）或 ≤ 20¢（NO策略）
3. 剩余时间 > 6小时（有足够兑现时间）

**早入场的容忍条件：**
- 距到期 >3天 但 VR ≥ 2.0 + 价格 ≤ 10¢ → 可小仓位入（≤10%）
