# 数据源与备份策略

> 核心原则：永远不依赖单一数据源。主源挂了，备源自动顶上。

---

## 一、推文数据源（最关键）

### 计数规则（与 Polymarket 保持一致）
- ✅ 计入：主推文（type=tweet, isReply=false）、Quote、Repost（RT）
- ❌ 不计：回复（isReply=true）
- 字段判断：`isReply === false` → 计入；`isReply === true` → 排除

---

### 数据源优先级

| 优先级 | 数据源 | 接口 | 密钥 | 稳定性 | 备注 |
|--------|--------|------|------|--------|------|
| 🥇 主源 | **twitterapi.io** | `GET /twitter/user/last_tweets?userName=elonmusk` | `new1_8452e6aed9cd49e9b163a11635102474` | ★★★★★ | 有 `isReply` 字段，精准过滤 |
| 🥈 备源1 | **xtracker.polymarket.com** | `/api/users/elonmusk/posts?limit=5` | 无需 | ★★★ | 官方源，但有间歇性 bug（返回空数组），不可单独依赖 |
| 🥉 备源2 | **polystrike.xyz** | 页面抓取 | 无需 | ★★★ | 社区备用计数器，免费，适合人工验证 |
| 🔄 备源3 | **RapidAPI xscraper** | `GET /user-tweets?username=elonmusk` | Vercel 环境变量 `RAPIDAPI_KEY` | ★★★ | 现有旧主源，偶发 429 |
| 🗃️ 降级 | **GitHub Gist 快照** | `https://gist.githubusercontent.com/Adul9981/d174b4498c408076ff218e164f24807e/raw/` | 无需 | ★★ | 最后兜底，数据可能滞后 |

## 二、twitterapi.io 接口详情

### 获取最新推文
```
GET https://api.twitterapi.io/twitter/user/last_tweets?userName=elonmusk
Header: X-API-Key: new1_8452e6aed9cd49e9b163a11635102474
```

**响应字段（关键）：**
```json
{
  "type": "tweet",
  "id": "...",
  "text": "...",
  "createdAt": "Mon Jun 01 04:58:38 +0000 2026",
  "isReply": false,
  "inReplyToId": null,
  "retweeted_tweet": {...},   // 非 null = 转推
  "quoted_tweet": {...}        // 非 null = 引用推文
}
```

### 获取用户信息（含总推文数）
```
GET https://api.twitterapi.io/twitter/user/info?userName=elonmusk
Header: X-API-Key: new1_8452e6aed9cd49e9b163a11635102474
```

### 按 ID 获取推文
```
GET https://api.twitterapi.io/twitter/tweet/ids?ids=xxx,yyy
Header: X-API-Key: new1_8452e6aed9cd49e9b163a11635102474
```
API 文档：https://docs.twitterapi.io/api-reference/endpoint/get_tweet_by_ids

## 三、xtracker 问题说明

- xtracker.polymarket.com 是 Polymarket 官方数据源
- **已知 bug**：5-6 月多次出现计数器卡住、返回空数组（`{"success":true,"trackings":[]}`）
- 社区反馈：polystrike.xyz、X 上大量用户吐槽
- **结论**：xtracker 不可单独依赖，必须有独立备源

---

## 四、切换逻辑（代码层面）

```
1. 先调 twitterapi.io → 成功则用
2. twitterapi.io 失败 → 调 xtracker.polymarket.com
3. xtracker 失败或返回空 → 调 RapidAPI xscraper
4. 全部失败 → 读 GitHub Gist 快照（标注数据滞后时间）
```
