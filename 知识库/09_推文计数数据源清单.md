# 马斯克推文计数数据源清单

> 记录所有尝试过的数据源，结论/状态持续更新

---

## ✅ 当前使用（优先级顺序）

### 1. polystrike.xyz（主力，2026-06-01 接入）
- **接口**：`GET https://polystrike.xyz/api/v1/meta/elon`
- **返回**：`real_counter`（社区抓取）、`polymarket_xtracker_counter`（官方）、每期 start_ts/end_ts
- **免费额度**：100 requests/hour，无需 API Key
- **优点**：最准，直接对齐 Polymarket 规则，无翻页限制
- **缺点**：第三方，无 daily breakdown（无法画每日柱状图）
- **状态**：✅ 正常，5月26–6月2期 real_counter=122

### 2. twitterapi.io（fallback）
- **接口**：`GET https://api.twitterapi.io/twitter/user/last_tweets?userName=elonmusk`
- **Key**：`new1_8452e6aed9cd49e9b163a11635102474`（写死兜底）
- **优点**：有每条推文时间戳，可算 daily breakdown 和热力图
- **缺点**：Vercel 10s 内最多翻 12 页（~240条），马斯克高发期数据截断，计数偏低
- **状态**：⚠️ 仅在 polystrike 失败时启用

## ❌ 已淘汰

### 3. xtracker.polymarket.com（官方，已弃用）
- **接口**：`GET https://xtracker.polymarket.com/api/users/elonmusk/trackings?activeOnly=true`
- **问题**：经常返回 0，不稳定，官方也在抢修
- **状态**：❌ 保留为市场元数据兜底（市场列表），不再用于推文计数

### 4. RapidAPI xscraper（已弃用）
- **接口**：`GET https://{RAPIDAPI_HOST}/user-tweets?username=elonmusk`
- **问题**：需要付费 Key，限流严，且计数与 Polymarket 规则不完全一致
- **状态**：❌ 已移除

### 5. twitterapi.io last_tweets（单页，已升级）
- 早期版本只拉1页（约20条），严重低估
- 已升级为多页翻页版本，但仍受 Vercel 超时限制

## 规则对齐说明

Polymarket 计数规则（以 oracle 为准）：
- **不计** isReply=true 的回复推文
- **计入** 转推（retweet）
- 时区以 UTC 为准（市场的 startDate/endDate 都是 UTC）
