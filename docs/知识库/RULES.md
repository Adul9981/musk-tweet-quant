## 一、理解这个市场的本质

### 1.1 这不是预测马斯克——这是预测市场的定价偏差

市场价格反映的是**所有参与者的集体认知**，不是真实概率。
你的优势来自于：**你有更精确的µ模型，而市场定价粗糙。**

> 价值比 = 模型估算的真实获胜概率 / 市场当前价格
> 价值比 > 1.0 → 市场低估了这个区间 → 有正期望值（EV > 0）
> 价值比 < 1.0 → 市场高估了这个区间 → 负期望值，不入场

只要长期执行正期望值的操作，就会盈利。这是数学，不是运气。

### 1.1.1 预测正确 ≠ 下注正确（重要！）

🔴 **铁律：中心落点是模型预测的最可能区间，但不一定是最值得买入的区间。**

两件事必须同时为真，才值得入场一个区间：
1. 模型认为这个区间有较高概率获胜
2. **市场给出的价格低于真实概率**（即价值比 ≥ 1.0）

**典型陷阱**：模型预测180-199最可能，市场价格44¢，但真实概率可能只有30-35%。
- 买入44¢，需要 >44% 真实概率才能盈利 → 负EV
- 两侧区间价格20¢，真实概率约20% → 接近EV盈亏平衡
- **不能因为「最可能」就买，必须看价格是否合理**

### 1.1.2 中心区间为什么经常被高估

**机制**：
- 散户凭感觉买「最可能」区间 → 资金涌入 → 价格被推高
- 做市商按公众情绪定价，不做精确计算
- 市场已充分定价的信息，没有套利价值

**规律**：当中心区间价格 **> 35¢** 时，需要警惕该区间是否已被过度定价。
此时两侧区间往往存在更好的盈亏比和价值比。——来源：回测分析 + 2026-05-27市场观察

### 1.2 市场的定价惰性

Polymarket 的做市商更新价格有滞后，尤其是：
- **BJ 12:00 前后**：市场还没定价即将到来的深夜爆发，这是最大的定价洼地
- **当日异常发生后**：市场过度反应，中心区间价格短暂失真
- **到期前最后24小时**：流动性下降，价差扩大，但高确定性区间仍有操作空间

### 1.3 你在和谁竞争

参与这个市场的人大致分三类：
- **散户**：凭感觉买，买热门区间，不看价值比
- **数据玩家**：有模型，会看pace，但不一定有时间规律优势
- **做市商**：赚价差，不押方向

你的优势在于：**206天时间规律数据 + 会话模型**，大多数参与者没有这个。

## 二、µ预测规则

### 2.1 最佳µ优于简单µ

🟡 始终使用**最佳µ**（时间加权µ 和 会话修正µ 的均值）做区间判断，而非 C + R×T

简单µ的误差来源：
- 忽略了剩余时间内各小时的发推权重不同
- 没有考虑今日会话是否缺席

### 2.2 µ的不确定性随时间增大

| 剩余时间 | µ的不确定性（约1σ） | 含义 |
|---------|-----------------|------|
| 3天 | ±50条 | 区间判断不可靠，不要入场 |
| 2天 | ±35条 | 开始有参考价值 |
| 1天 | ±20条 | 较可靠，适合入场 |
| 12小时 | ±12条 | 高可信，可加仓 |
| 6小时 | ±8条 | 非常可信 |

🔴 **铁律**：到期前超过3天不入场，µ误差太大，不是投资，是赌博。

### 2.3 会话缺席的µ影响

今日已过时段若缺席，对µ的影响：
- 深夜会话缺席（BJ 13-16无推文）→ µ 下移约 **14条**
- 下午会话缺席（BJ 0-5异常低）→ µ 下移约 **14条**
- 上午会话缺席 → µ 下移约 **11条**

🟡 每天BJ 17:30死区评估时，必须检查已完成的会话是否正常。

## 三、入场规则

### 3.1 最佳入场时间窗口

🔴 **铁律：到期前 2–2.5 天入场，不早不晚**

| 入场时机 | 评级 | 原因 |
|---------|------|------|
| BJ 12:00–12:30（深夜爆发前） | 🥇 最强 | 市场未定价深夜爆发，价格洼地 |
| BJ 21:00–21:30（晨间窗口） | 🥈 次强 | 美国上午开始，活跃度回升 |
| 死区之后（BJ 19:00-20:00） | 🥉 可用 | µ清晰，情绪稳定 |
| 活跃高峰中（BJ 13-16） | ⚠️ 慎用 | 价格已被推高，价值比下降 |
| 到期前 < 1天 | ❌ 风险高 | 波动剧烈，除非非常确定 |

### 3.2 入场条件检查清单

入场前必须确认（**三条都满足**才入场）：
- [ ] 距到期 **2–2.5 天**以内
- [ ] 目标区间价值比 **≥ 1.0**（最好 ≥ 1.2）
- [ ] 当前时机不是活跃高峰（避免追高）

**增加第四步——价值比横向比较**：
- [ ] 对比中心区间和两侧相邻区间的价值比，选择价值比**最高**的区间入场
- 如果中心区间价格 > 35¢，必须显式比较两侧区间是否更划算

### 3.3 各区间价值比评估方法

入场前对每个候选区间估算价值比：

| 区间类型 | 模型概率估算 | 参考阈值 | 建议 |
|---------|------------|---------|------|
| 中心落点区间 | 25-40%（回测中位） | 价格 ≤ 30¢ | 主仓候选 |
| 中心±1档邻区 | 15-25% | 价格 ≤ 20¢ | 主仓候选或保护仓 |
| 中心±2档 | 5-15% | 价格 ≤ 10¢ | 小仓高赔率候选 |
| 中心±3档及以外 | <5% | 价格 ≤ 5¢ | 尾部押注，≤5%仓位 |

🟡 **原则**：在同等模型信心下，优先买更便宜（价值比更高）的区间，而非盲目买「中心落点」。

### 3.4 边界分仓规则

🟡 若最佳µ距某区间边界 **≤ 10条**，不要把全部仓位压一个区间：
- 各50%分在两侧区间
- 理由：µ本身有±10条的预测误差，边界两侧都有实质概率

🟡 **µ不确定性两侧覆盖**：µ不确定性 > 20条时（剩余时间 > 1.5天），中心落点两侧都要有仓位覆盖，不能只押一侧。单侧重仓 + µ预测出错 = 无对冲。——来源：5月26日复盘（详见4.1仓位结构）

## 四、仓位管理规则

### 4.1 仓位集中原则（含两档覆盖策略）

🔴 **铁律：主仓集中，不分散买5个区间**

分散买5个区间的问题：
- 边缘区间大概率亏损
- 边缘亏损会吃掉中心区间的盈利

**正确结构（三层）：**

| 仓位 | 比例 | 区间 | 条件 |
|------|-----|------|------|
| 主仓 | 50–70% | 价值比最高的区间（通常是中心落点或其相邻区间） | 价值比 ≥ 1.0 |
| 保护仓 | 20–30% | 中心落点**下方1档**邻区 | 模型有偏高倾向（+0.3档系统性偏差），下侧风险更大 |
| 高赔率仓 | ≤10% | 价值比 ≥ 2.5 的低价区间 | 可选，有合适目标时加 |

**为什么要加保护仓（来自回测数据）：**
- 33期回测显示模型存在 **+0.3档的系统性偏高**
- t-1时±1档命中率85%（近期），意味着预测区间或其相邻区间获胜概率极高
- 只押中心1档 → 命中率38%；覆盖中心+下方1档 → 覆盖率约65%+
- 成本：两档仓位平摊了资金，单档收益降低，但整体EV显著提升

🟡 **何时不需要保护仓**：模型µ与中心区间边界相差 > 15条，且当期节奏平稳（偏差 < ±15%）。此时中心落点获胜概率较高，可以适当减少保护仓比例。

### 4.2 高赔率补充仓规则

盈亏比和期望值是两套互补的框架，缺一不可：

| 框架 | 核心问题 | 何时用 |
|------|---------|-------|
| **期望值** | 平均下来赚还是亏？ | 评估主仓是否值得建 |
| **盈亏比** | 赢一次能弥补多少次输？ | 评估小仓位尾部押注 |

**盈亏比公式：**
- 盈亏平衡概率 = 买入价 / 赔付价
- 例：1.7¢买入，赔付100¢ → 盈亏平衡概率 = 1.7%（只需真实概率>1.7%即正期望）

🟡 满足以下**全部条件**时，用小仓位（≤10%利润）押高赔率区间：
- 盈亏比 ≥ 20:1（买入价 ≤ 5¢）
- **你相信真实概率 > 盈亏平衡概率**（不能只看赔率高就买）
- 模型概率与市场定价的差距在合理范围（价值比 0.5x 以上）
- 已有主仓在盈利中（用利润博，不用本金博）

⚠️ **高盈亏比的陷阱：**
- 58:1的盈亏比听起来很诱人，但若真实概率只有0.03%，仍是负期望
- 泊松模型低估尾部事件（马斯克发推有突发性），真实概率可能略高于模型
- 已有该区间仓位时，再加仓前先问：「我已经有这个暴露了，额外加仓边际价值是什么？」

> 历史案例：5月12日 100–119 区间，价值比=3.0，市价5¢，没买，最终涨到85¢（17倍）。
> 这种机会每期都可能出现，错过一次就是错过数倍收益。

### 4.3 调仓规则

🟡 **µ偏移超过 1.5σ（约25-30条）时才考虑调仓**，不要每天重算换仓。

频繁调仓的问题：
- 每次换仓都有买卖价差损耗
- 短期µ波动是噪音，不是信号
- 增加执行难度和心理负担

调仓时机只有两个：
1. **死区（BJ 17:30–19:00）**：最理性，µ稳定，不受活跃期情绪干扰
2. **确认µ大幅偏移**（超过1.5σ），不是"感觉可能要变"

🔴 **连续降温触发规则**：Musk连续2天发推量低于历史均值30%+，视为µ下修信号，必须在当天死区内重算µ并评估调仓。不能等「再看一天」。——来源：5月26日复盘

## 五、出场规则

### 5.1 止盈窗口

🟡 **BJ 14:00–15:00（深夜高峰峰值）是最佳止盈评估时间**

- 全天推文速率最高，µ此时最接近真实终点
- 市场此时价格最高（中心区间可能已到70–80¢）
- 如果已达目标收益，可考虑部分止盈

### 5.2 止损与剪仓

🔴 **铁律：死区（BJ 17:30–19:00）是唯一强制执行剪仓的窗口**

> 来自5月26日复盘的血泪教训：
> 昨天已判断落点要变，但早上看到差距拉大，越拖越不想动。
> 正确做法：在死区（价差还小、情绪最平静时）执行，不等次日。

死区剪仓原则：
- 对概率 < 15% 的区间 → 强制评估，大概率应该出
- 浮亏 > 20% 且µ已偏出区间 → 接受损失，止损出场
- 「再等等看」= 死亡陷阱，死区过后下一个活跃期会让你更难出场

### 5.3 到期前最后操作

🔵 到期前 6–12 小时，如果中心区间价格已达 75¢ 以上：
- 可以选择锁定部分利润（卖出50%）
- 剩余50%博到期全额（$1）

## 六、执行纪律规则

### 6.1 不追节奏

🔴 **铁律：马斯克当日发推异常，不临时改变主仓位**

发推异常有两种：
- **异常多**：市场会推高所有区间价格，追买等于追高
- **异常少**：市场恐慌，但单日异常不改变整期µ（206天数据：偶尔的沉默日不影响整体节奏）

唯一需要调整的信号：µ连续2天偏离超过1.5σ。

### 6.2 不依赖主观判断

🟡 所有操作必须提前设定条件，对照条件执行，不临场决策：

> 对的方式：「如果价值比≥1.2且剩余时间在2天内，执行建仓」
> 错的方式：「感觉今天节奏不对，再等等看」

实时主观判断的失败率极高（见4月21日复盘），条件触发式操作更稳定。

### 6.3 每日三个强制检查点

| 时间 | 检查内容 | 行动 |
|------|---------|------|
| **BJ 11:45** | 当前µ是否支持建仓？价值比是否合适？ | 符合条件→立即建仓 |
| **BJ 17:30** | 现有仓位µ是否偏移？有无需要剪的仓？ | 有问题→当场解决，不推迟 |
| **BJ 21:00** | 晨间窗口是否有补仓机会？ | 符合条件→建仓或补仓 |

**这三个时间点的提醒是强制的，不是可选的。**

## 八、反面教材（绝对不要做的事）

| ❌ 错误行为 | 后果 | 正确做法 |
|-----------|------|---------|
| 提前3天以上入场 | 噪音极大，被虚晃 | 到期前2–2.5天 |
| 分散买5个区间 | 边缘亏损吃掉中心盈利 | 集中买1-2个区间（主仓+保护仓） |
| 在活跃高峰追买 | 追高，价值比已下降 | BJ 12:00前建仓 |
| 看到差距拉大就拖延调仓 | 越拖越难出 | 死区强制执行 |

| 当日节奏异常就改主仓 | 被短期噪音带跑 | 看µ偏移量，不看单日 |
| 空仓等"完美时机" | 永远等不到 | 价值比≥1.0即可入场 |
| 亏损区间继续加仓 | 把小亏变大亏 | 加仓唯一依据：µ支持这个区间 |

| **只看方向，不看价格** | 长期来看必然负EV | 价值比 < 1.0 的区间坚决不入场 |

## 九、认知偏差速查（每次操作前对照）

> 每条是一个陷阱名 + 一条规则。来自真实失误，不是理论。

| 陷阱 | 触发场景 | 🔴 规则 |
|------|---------|--------|
| **距离幻觉** | "只差X条了，感觉快赢了" | 「近」不是概率。区间边界高溢价=卖出信号，不是持仓信号 |
| **叙事屈服** | 听到反驳后直觉上想改立场 | 任何改变判断的理由，先跑数字再开口，不能先表态 |
| **亏损追仓** | "用赚的钱赌一把弥补亏损" | 盈利仓和亏损仓是两笔独立交易，不能互相抵消 |
| **概率不自洽** | 认为市场错了，但用市场概率算EV | 认为市场错 → 必须用模型概率算EV，不能混用 |
| **拖延效应** | 差距越大越不想动仓 | 调仓只看µ是否支持，不看浮亏多少。死区是最后窗口 |
| **已有暴露** | "这个区间值得押，我要加仓" | 先问：已有仓位是否已覆盖这个暴露？额外加仓边际价值是什么？ |
| **方向正确幻觉** | "我押对了方向（中心落点），结果还是亏" | 预测正确 ≠ 下注正确。必须同时满足：概率正确 + 价格合理（价值比≥1.0） |

## 十、套利技巧

### 10.1 深夜爆发前定价洼地（最核心）

**机制**：BJ 12:00 活跃度只有 1.2条/h，市场按当前速率定价，没有为即将到来的深夜爆发（3.0条/h）定价。

**操作**：BJ 11:45–12:15 建仓，中心区间此时价格最低，BJ 14–15 高峰评估止盈。

**量化**：历史上 BJ 12→13 的跳跃幅度约 +150%，是全天最大的单小时跳跃。

---

### 10.2 中心区间定价过高时，转买两侧（核心策略）

**机制**：散户持续买入「最可能」区间，将中心区间价格推高到真实概率以上。
两侧相邻区间因无人关注，往往被低估。

**触发条件**（同时满足）：
1. 中心落点区间价格 **> 35¢**
2. 模型估算真实概率 **< 市场价格**（价值比 < 1.0）
3. 两侧区间价格 **< 25¢**，且模型估算真实概率 > 价格

**操作**：
- 放弃或轻仓中心区间
- 主仓移向价值比最高的相邻区间（通常是下方1档，因模型偏高+0.3档）
- 可同时在中心区间上/下各建一个相邻区间仓位（双侧覆盖）

**实例（2026-05-27观察）**：
| 区间 | 市场价 | 估算真实概率 | 价值比 |
|------|-------|------------|--------|
| 180-199 | 44¢ | ~32% | 0.73 ❌ |
| 160-179 | 22.9¢ | ~22% | 0.96 ➖ |
| 200-219 | 20.5¢ | ~20% | 0.98 ➖ |
| 220-239 | 6.5¢ | ~10% | 1.54 ✅ |
| 140-159 | 3.2¢ | ~5% | 1.56 ✅ |

→ 中心区间明显负EV；两侧低价区间价值比更高，是真正的买入机会。

---

### 10.3 市场情绪溢价反向套利

**机制**：市场参与者受"距离感"影响，会对接近区间边界的区间大幅溢价。

**识别**：若某区间价格 > 模型概率 × 1.5（价值比 < 0.67），且你的µ已明显偏离该区间 → 是卖出时机而非持仓时机。

**操作**：在情绪高点卖出被高估区间，用资金转移到价值比 ≥ 1.2 的区间。

---

### 10.4 到期前最后6小时的高确定性套利

**机制**：最后6小时µ误差 ≤ ±8条，中心区间高度确定，但流动性下降导致价差扩大。

**操作**：
- 若中心区间 < 75¢，且模型概率 > 85% → 仍有正期望，可加仓
- 若中心区间 > 85¢ → 锁定50%利润，剩余博到期$1

---

### 10.5 高赔率小仓技巧（尾部暴露）

**机制**：泊松模型低估马斯克的突发性（肥尾），低概率区间的真实概率可能高于模型。

**进场条件**（同时满足）：
1. 市价 ≤ 5¢（盈亏比 ≥ 20:1）
2. 真实概率估计 > 市价（盈亏平衡概率）
3. 已有主仓在盈利中（用利润押，不用本金）
4. 当前仓位**没有**该区间的暴露（避免重复覆盖）

**仓位上限**：单笔小仓 ≤ 当期浮盈的 10%

---

### 10.6 多期同时操作（跨期分散）

**机制**：Polymarket 同时开多个到期日的市场，可以把资金分散在不同到期日上，降低单期异常的影响。

**策略**：
- 主仓放在到期 1.5–2 天的市场（µ最可信）
- 次仓放在到期 3–4 天的市场（提前布局，仓位较轻）
- 两期同时盈利时，用次期利润继续滚动建仓

## 十一、规则进化机制

> 规则库不是静态文档，每期结束后都应该进化一次。

### 11.1 什么值得写进规则库

满足以下任意一条的观察，值得沉淀：
- 🔁 **可复现**：在多期都能验证的规律，不是单次偶然
- 💰 **有实际盈亏影响**：能明显影响操作决策的
- 🧠 **反直觉**：和常识相反、容易犯错的认知偏差
- 📐 **可量化**：有具体的数字、概率或阈值

不值得写进来的：
- 单次运气性结果
- "感觉应该"但没有数据支撑的
- 已有规则的重复描述

### 11.2 进化触发条件（由 Claude 定期执行）

每期市场结束后，运行 `/musk-evolve` 触发以下检查：
1. 本期是否出现了新的认知偏差或操作失误？
2. 现有规则是否有被本期数据证伪的？
3. 套利技巧是否有新的模式值得补充？
4. 规则库是否有过时或矛盾的条目？

### 11.3 进化原则

- **加法有门槛**：新规则必须满足 11.1 的至少一条标准
- **删法更严格**：删除一条规则需要至少3次反例，或明确的逻辑推翻
- **不追求完美**：规则库宁可少而精，不要多而杂
- **来源标注**：每条规则注明来源（数据分析 / 复盘 / 高手操作）
